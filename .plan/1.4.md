## Why those rows show "Unknown"

The 12 customers showing "Unknown" are **not in the Partner Center customer list at all** — the API returns a genuine 404 for them. These are customers that have a GDAP relationship but were never transacted through this partner account (e.g., the customer set up the GDAP invite themselves, or the relationship was created through a different channel). The Partner Center API only knows about customers that were provisioned or purchased through this partner.

There is no domain available for them through any accessible API:
- `api.partnercenter.microsoft.com/v1/customers/{id}` → **404**
- Microsoft Graph delegatedAdminCustomers → **403** (no cross-tenant permission)
- Public AAD OIDC endpoints → only reflect the tenantId back, never the domain

The only data available for these 12 is their `tenantId` and `displayName` from the GDAP API — which is already what the page shows in the Name column anyway.

## The fix: show displayName as fallback instead of "Unknown"

The extension should pre-load the GDAP customer list (which it likely already does, or can do cheaply since that token is already in use) and use `displayName` as a fallback when the Partner Center API returns 404. Here's what to tell Claude:

---

For the customers that return 404 from `api.partnercenter.microsoft.com/v1/customers/{id}` (these are non-transacted GDAP-only customers — 12 out of 83), no domain name exists in any accessible API. Instead of showing "Unknown", fall back to showing the `displayName` from the GDAP customer list, styled in italic to visually distinguish it from real domain names.

**To do this:**

1. Before or alongside the Partner Center API call, fetch the full GDAP customer list to build a `displayNameMap`:
   ```
   GET https://api.partnercustomersecurity.microsoft.com/CustomerServiceAdminApi/Web/v1/delegatedAdminGdapCustomers?$orderby=displayName&$count=true
   Authorization: Bearer {sessionStorage.getItem('CustomerSvcAdminKey')}
   ```
   Build: `displayNameMap[customer.tenantId] = customer.displayName.trim()`

2. When a tenantId is missing from the Partner Center domain map (404 or absent), look it up in `displayNameMap` and show it in italic, e.g.: `<em>Northwind Traders</em>`

3. Only show "Unknown" as a last resort if neither map has the tenantId.

4. The GDAP token (`CustomerSvcAdminKey`) is already in `sessionStorage` — no new auth needed. Since this fetch also goes cross-origin to `api.partnercustomersecurity.microsoft.com`, it must also go through the background service worker. Add that domain to `host_permissions` in `manifest.json`:
   ```json
   "host_permissions": [
     "https://partner.microsoft.com/*",
     "https://api.partnercenter.microsoft.com/*",
     "https://api.partnercustomersecurity.microsoft.com/*"
   ]
   ```
   And add a second message handler in `background.js` for `FETCH_GDAP_CUSTOMERS` that uses `CustomerSvcAdminKey` as the bearer token.
