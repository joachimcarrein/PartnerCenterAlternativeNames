## Chrome Extension Prompt: Microsoft Partner Center — Tenant Name Column

---

You are building a Chrome extension that adds an extra **"Tenant Name"** column to the customer list table on the Microsoft Partner Center Granular Administration page.

---

### Target Page

**URL pattern:** `https://partner.microsoft.com/dashboard/v2/customers/granularadminaccess/*`

The page shows a paginated list of customers with two columns: **Name** and **Microsoft ID** (which is the customer's Azure AD tenant GUID).

---

### DOM Structure (critical — the page uses Web Components / Shadow DOM)

The table lives inside a custom element:
```
<CUSTOMERSVCADMIN_HE-DATA-GRID>
  #shadow-root
    <table class="data-grid__table he-table he-table--hover" role="grid">
      <tr id="column-header">
        <th id="header-selection" ...>
        <th id="cell-displayName-column-header">Name</th>
        <th id="cell-tenantId-column-header">Microsoft ID</th>
      </tr>
      <!-- 10 data rows per page -->
      <tr role="row" id="row-{tenantId}" class="data-grid__row">
        <td id="cell-selection-{tenantId}" ...>
        <th id="cell-displayName-{tenantId}" class="data-grid__row-header">
          <span class="data-grid__row-header__content ...">
            <span>{customerName}</span>   <!-- the clickable link is inside here -->
          </span>
        </th>
        <td id="cell-tenantId-{tenantId}" class="data-grid__cell">
          {tenantId-guid}
        </td>
      </tr>
    </table>
</CUSTOMERSVCADMIN_HE-DATA-GRID>
```

Key facts:
- The table is in a **shadow root** — you cannot query it with `document.querySelector`. You must traverse the shadow DOM to reach it.
- The **Name** column is a `<th>` (not `<td>`), with `id="cell-displayName-{tenantId}"`.
- The **Microsoft ID** column is a `<td>` with `id="cell-tenantId-{tenantId}"`.
- Each row's `id` is `row-{tenantId}`, so the tenantId is trivially extractable from any row or cell ID.
- The page shows **10 rows per page**. Pagination and the **search bar filter rows without a full page reload** — the DOM is mutated in place by the web component. You must use a **`MutationObserver`** on the shadow root's table (or the `<CUSTOMERSVCADMIN_HE-DATA-GRID>` host element) to detect when rows change and re-inject your column.
- Finding the shadow host: traverse `document.querySelectorAll('*')` and look for an element whose `tagName` includes `'DATA-GRID'`, then access `.shadowRoot`. Be aware this element may itself be nested inside other shadow roots — use a recursive shadow DOM traversal helper function.

---

### API to Fetch Tenant Names

**Use the `delegatedAdminGdapCustomers` endpoint** — it already returns `displayName` for every customer and is the same call the page itself makes. Do **not** use `delegatedAdminRelationships` for names, because some relationships store old/renamed tenant names that differ from the current GDAP name.

**Endpoint:**
```
GET https://api.partnercustomersecurity.microsoft.com/CustomerServiceAdminApi/Web/v1/delegatedAdminGdapCustomers?$orderby=displayName&$count=true
```

**Authentication:** The page stores a short-lived JWT in `sessionStorage` under the key `CustomerSvcAdminKey`. Read it at runtime:
```js
const token = sessionStorage.getItem('CustomerSvcAdminKey');
// Use as: Authorization: Bearer {token}
```

This token is scoped to `https://api.partnercustomersecurity.microsoft.com` and will return 401 if expired. Handle expiry gracefully by showing a placeholder and retrying on the next page visit.

**Fetch strategy — fetch ALL records upfront, not per-row.** Rationale:
- The endpoint returns all 83 (or however many) customers in a single request — it's fast and lightweight.
- Since search/filtering updates the DOM without reloading, having the full dataset already in memory means you can instantly annotate any row that appears without additional network round-trips.
- Fetching only visible rows would require repeated API calls on every search keystroke.

**Fetch on:** page load (once). Store results in memory as a `Map<tenantId, displayName>`. Also persist to `chrome.storage.local` with a timestamp for caching (see below).

---

### Caching Strategy

- Cache the `Map<tenantId, displayName>` in `chrome.storage.local` as a JSON object, alongside a `cachedAt` timestamp (ISO string).
- Cache key: `gdapTenantNames`
- Expiry: **30 days** (`30 * 24 * 60 * 60 * 1000` ms).
- On load: read from cache first. If valid (not expired), use it immediately. If expired or missing, fetch fresh from the API and update the cache.
- Since the token in sessionStorage is short-lived, always fetch the token fresh from `sessionStorage` at the time of the API call — never cache the token itself.

---

### Column Injection Logic

1. **Find the shadow root** of `CUSTOMERSVCADMIN_HE-DATA-GRID` (recursive traversal needed).
2. **Add a header cell** to the `<tr id="column-header">` row after the `cell-tenantId-column-header` `<th>`:
   ```html
   <th id="cell-tenantName-column-header" role="columnheader" class="data-grid__table-header">Tenant Name</th>
   ```
3. **For each data row** `<tr role="row">`, extract the `tenantId` from the row's `id` attribute (`row.id.replace('row-', '')`), then append a new `<td>` after the `cell-tenantId-*` cell:
   ```html
   <td id="cell-tenantName-{tenantId}" class="data-grid__cell">
     <span class="data-grid__cell-content-wrapper">{tenantName or 'Loading...'}</span>
   </td>
   ```
4. **On MutationObserver trigger** (rows changed): re-run step 3 for any rows that don't already have your injected cell (`id="cell-tenantName-*"`). Also re-check the header (it may be re-rendered too).
5. Guard against double-injection: before adding a header or row cell, check if it already exists by ID.

---

### MutationObserver Setup

```js
const observer = new MutationObserver(() => {
  injectTenantNameColumn(shadowRoot, tenantMap);
});
observer.observe(shadowRoot, { childList: true, subtree: true });
```

Observe the **shadow root itself** with `subtree: true` so you catch row additions at any depth. Debounce the callback by ~100ms to avoid excessive re-runs during rapid DOM mutations.

### Error Handling

- **Token missing or expired (401):** Show `"—"` in the tenant name cell, log a warning, retry on next page navigation.
- **Network error:** Show `"Error"` in cells, log to console.
- **Shadow root not yet available:** Retry with `setTimeout` up to 10 times at 500ms intervals before giving up.
- **Row has no matching tenantId in map:** Show `"Unknown"`.

### Debugging Instructions

When something goes wrong, follow this process:

**1. Verify the shadow DOM is being found:**
```js
// Run in DevTools console on the Partner Center page:
const findGrid = (root, depth = 0) => {
  if (depth > 8) return null;
  for (const el of root.querySelectorAll('*')) {
    if (el.tagName && el.tagName.includes('DATA-GRID')) return el;
    if (el.shadowRoot) {
      const found = findGrid(el.shadowRoot, depth + 1);
      if (found) return found;
    }
  }
  return null;
};
const grid = findGrid(document);
console.log('Grid element:', grid);
console.log('Shadow root:', grid?.shadowRoot);
console.log('Rows:', grid?.shadowRoot?.querySelectorAll('tr[role="row"]:not(#column-header)').length);
```

**2. Verify the auth token:**
```js
const token = sessionStorage.getItem('CustomerSvcAdminKey');
console.log('Token exists:', !!token, 'Length:', token?.length);
// Token should be a long JWT string (500+ chars)
```

**3. Test the API call manually:**
```js
const token = sessionStorage.getItem('CustomerSvcAdminKey');
const r = await fetch('https://api.partnercustomersecurity.microsoft.com/CustomerServiceAdminApi/Web/v1/delegatedAdminGdapCustomers?$orderby=displayName&$count=true', {
  credentials: 'include',
  headers: { 'Authorization': 'Bearer ' + token }
});
console.log('Status:', r.status);
const data = await r.json();
console.log('Count:', data['@odata.count'], 'First item:', data.value?.[0]);
```

**4. Check the cache:**
```js
chrome.storage.local.get('gdapTenantNames', (result) => {
  const cache = result.gdapTenantNames;
  console.log('Cached at:', cache?.cachedAt);
  console.log('Entries:', Object.keys(cache?.data || {}).length);
  const age = Date.now() - new Date(cache?.cachedAt).getTime();
  console.log('Age (days):', (age / 86400000).toFixed(1));
});
```

**5. Check if injection ran:**
```js
const findGrid2 = (root, d=0) => { if(d>8) return null; for(const el of root.querySelectorAll('*')) { if(el.tagName?.includes('DATA-GRID')) return el; if(el.shadowRoot){const f=findGrid2(el.shadowRoot,d+1);if(f)return f;} } return null; };
const sr = findGrid2(document)?.shadowRoot;
console.log('Header injected:', !!sr?.querySelector('#cell-tenantName-column-header'));
console.log('Row cells injected:', sr?.querySelectorAll('[id^="cell-tenantName-"]').length);
```

**6. Force a cache clear and refetch:**
```js
chrome.storage.local.remove('gdapTenantNames', () => console.log('Cache cleared — reload the page to refetch'));
```

**7. Common problems and fixes:**
- *Column appears but disappears on search:* MutationObserver is not watching `subtree: true`, or the debounce is too aggressive — reduce debounce or remove it.
- *Column never appears:* Shadow DOM traversal is failing. Add `console.log` at each depth level of the traversal. The `DATA-GRID` element may be nested inside another shadow root.
- *All cells show "Loading..." forever:* API call is silently failing. Check the network tab for the API request, or run debug step 3 above.
- *Token returns 401:* The `CustomerSvcAdminKey` in sessionStorage has expired (it's short-lived). The user needs to navigate away and back, or refresh the page. The extension cannot renew it — it can only read what the page puts there.
- *Injected column styling is misaligned:* Copy the exact `class` names from adjacent cells (`data-grid__cell`, `data-grid__table-header`) and match their inline styles if needed.

### Notes on Known Data Quirks

- Some customers have a leading space in their `displayName` (e.g. `" Northwind Traders"`). Always `.trim()` display names before showing them.
- 5 out of 83 customers in the sample data have slightly different names between the GDAP list and the relationships API — use the GDAP list names as the source of truth, since that's what this page already shows.
- The total customer count in the sample is 83, spread across 9 pages of 10 rows each.
