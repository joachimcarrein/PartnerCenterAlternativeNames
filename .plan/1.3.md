### Issue 2: CORS blocks the content script's fetch (`Failed to fetch`)

This is the real bug in the 5:47 PM "fixed" build. The token is fine, the URL is right, but the fetch still fails. Here's why:

When a content script calls `fetch()` to a cross-origin URL, Chrome sends the request with `Origin: chrome-extension://fkjaokhnjkkoiocblcgiikajaffdmbci`. The `api.partnercenter.microsoft.com` API has **no CORS headers** — it relies on the browser's same-site cookie credentials and expects `Origin: https://partner.microsoft.com`. It rejects the extension origin → browser throws `TypeError: Failed to fetch`.

`host_permissions` in the manifest grant the **service worker** cross-origin access, but content scripts still face CORS restrictions when they call `fetch()` directly.

**The fix: move the fetch to the background service worker.** The content script asks the background to do the fetch via `chrome.runtime.sendMessage`, which bypasses CORS entirely. Here is the exact change needed:

**Add `background.js`:**
```javascript
chrome.runtime.onMessage.addListener((msg, sender, sendResponse) => {
  if (msg.type !== 'FETCH_CUSTOMERS') return false;
  
  fetch(`https://api.partnercenter.microsoft.com/v1/customers?size=300`, {
    headers: {
      'Authorization': `Bearer ${msg.token}`,
      'Accept': 'application/json'
    }
  })
  .then(r => r.json())
  .then(data => sendResponse({ ok: true, data }))
  .catch(err => sendResponse({ ok: false, error: err.message }));
  
  return true; // keep channel open for async response
});
```

**Update `manifest.json`** — add the background service worker:
```json
{
  "background": {
    "service_worker": "background.js"
  }
}
```

**Update `content.js`** — replace the direct `fetch()` call inside `fetchAllDomains()` with:
```javascript
// Instead of: const resp = await fetch('https://api.partnercenter.microsoft.com/...')
// Use:
const result = await new Promise((resolve, reject) => {
  chrome.runtime.sendMessage({ type: 'FETCH_CUSTOMERS', token }, (response) => {
    if (chrome.runtime.lastError) return reject(new Error(chrome.runtime.lastError.message));
    if (!response.ok) return reject(new Error(response.error));
    resolve(response.data);
  });
});
// result.items is your customers array, result.continuationToken for pagination
```

For pagination, pass the `continuationToken` back through the message and handle it in `background.js` with the `MS-ContinuationToken` header.
