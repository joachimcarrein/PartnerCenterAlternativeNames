## Prompt: Update Chrome Extension — Show Tenant Domain on Microsoft Partner Center GDAP Page

### Context

I have an existing Chrome extension that adds a "Tenant Name" column to the Microsoft Partner Center GDAP customer list page (`https://partner.microsoft.com/dashboard/v2/customers/granularadminaccess/list`). The extension currently fetches the wrong data. Your task is to update it so it shows the **primary domain** (e.g., `acmecorp.onmicrosoft.com`) for each customer.

---

### What Needs to Change

The extension currently uses the **GDAP API** (`api.partnercustomersecurity.microsoft.com`) to get a `displayName`. That is incorrect. The correct approach is:

- **Use the Partner Center Customer API** (`api.partnercenter.microsoft.com/v1/customers`) to get the primary domain
- The field to display is **`companyProfile.domain`** (e.g., `acmecorp.onmicrosoft.com`)
- Rename the column header from "Tenant Name" to **"Tenant Domain"**

---

### Correct API Details

#### Auth Token (CRITICAL — different from the old token)

```javascript
const authData = JSON.parse(sessionStorage.getItem('AuthContextData'));
const token = authData.tokenMetadata.accountsFirstPartyApp.accessToken;
```

This token is scoped to `https://api.partnercenter.microsoft.com`. Do NOT use `CustomerSvcAdminKey` for this API — it will return 401.

#### Fetch All Customers (Paginated List)

```
GET https://api.partnercenter.microsoft.com/v1/customers?size=300
Authorization: Bearer {token}
Accept: application/json
MS-RequestId: {random-guid}
MS-CorrelationId: {random-guid}
```

Response shape:
```json
{
  "totalCount": 83,
  "items": [
    {
      "id": "a1b2c3d4-0001-4abc-8def-111122223333",
      "companyProfile": {
        "tenantId": "a1b2c3d4-0001-4abc-8def-111122223333",
        "domain": "acmecorp.onmicrosoft.com",
        "companyName": "Acme Corporation"
      },
      "customDomains": ["acme.com", "acme.net"]
    }
  ],
  "continuationToken": "..."
}
```

**Pagination:** If `continuationToken` is present in the response, make a subsequent request with header `MS-ContinuationToken: {continuationToken}` and merge results. Repeat until no `continuationToken` is returned.

#### Fetch Single Customer (fallback / on-demand)

```
GET https://api.partnercenter.microsoft.com/v1/customers/{tenantId}
Authorization: Bearer {token}
```

Response includes the same `companyProfile.domain` and `customDomains` fields.

#### Domain Priority Logic

Use `companyProfile.domain` as the primary value. This is always present and is the onmicrosoft.com domain (e.g., `acmecorp.onmicrosoft.com`). You may optionally also show `customDomains[0]` if present, but `companyProfile.domain` must always be shown — it's the stable identifier the user relies on.

### Unchanged Requirements

These remain exactly as before:

#### Target Page
`https://partner.microsoft.com/dashboard/v2/customers/granularadminaccess/list`

#### DOM Structure (Shadow DOM)

The entire table lives inside a Shadow DOM. Use recursive traversal:

```javascript
function findShadowRoot(root) {
  if (root.shadowRoot) return root.shadowRoot;
  for (const child of root.children) {
    const found = findShadowRoot(child);
    if (found) return found;
  }
  return null;
}
const shadowHost = document.querySelector('CUSTOMERSVCADMIN_HE-DATA-GRID');
const shadow = shadowHost.shadowRoot;
```

Row and cell IDs (inside the shadow root):
- Each row: `<tr role="row" id="row-{tenantId}">`
- Name cell: `<th id="cell-displayName-{tenantId}">` ← NOTE: `th` not `td`!
- Microsoft ID cell: `<td id="cell-tenantId-{tenantId}">`

To inject a new column cell into a row, insert a new `<td>` after the existing cells. The column header row is `<tr id="column-header">` — add a `<th>` there.

#### MutationObserver (Search Without Page Reload)

The page's search/filter updates the DOM without navigating. You MUST use a `MutationObserver` on the shadow root's table body to detect row additions/removals and inject the domain cell into newly rendered rows. Do not assume the DOM is stable.

#### Caching (30-Day Expiry)

Store the domain map in `chrome.storage.local` as:
```json
{
  "domainCache": {
    "a1b2c3d4-0001-4abc-8def-111122223333": "acmecorp.onmicrosoft.com",
    "..."
  },
  "domainCacheExpiry": 1751234567890
}
```

On load: if `Date.now() < domainCacheExpiry`, use the cached map. Otherwise, re-fetch and reset `domainCacheExpiry = Date.now() + (30 * 24 * 60 * 60 * 1000)`.

#### Fetch Strategy

Fetch ALL customers upfront using the paginated list endpoint (not per-visible-row). Store results in the cache. This avoids N×API calls and is fast since the list endpoint returns all records in a few pages.

### Common Problems to Watch For

| Problem | Likely Cause | Fix |
|---|---|---|
| All domain cells blank | Token expired or wrong token key | Check `AuthContextData` is present in sessionStorage; log token prefix |
| Only first page of customers loaded | Pagination not implemented | Check `continuationToken` in response; loop until absent |
| Domain cells disappear after search | MutationObserver not watching the right node | Observe the table `<tbody>` inside the shadow root, not the document |
| `Cannot read shadowRoot` | Shadow root not yet attached when script runs | Wrap in a `waitForElement()` loop with `requestAnimationFrame` or `setInterval` |
| 401 from API | Using wrong token (`CustomerSvcAdminKey`) | Switch to `AuthContextData → tokenMetadata.accountsFirstPartyApp.accessToken` |
| Column header duplicated after SPA navigation | Extension injecting header multiple times | Guard with `if (shadow.querySelector('#tenant-domain-header')) return;` before injecting |
