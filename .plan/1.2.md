### Bug Report

The extension has **two bugs**:

---

### Bug 1 — CRITICAL: `manifest.json` missing `host_permissions` for Partner Center API

**Error:** `TypeError: Failed to fetch` on every load.

**Cause:** `fetch()` from a content script to a cross-origin URL requires the target domain to be declared in `host_permissions` in `manifest.json`. Without it, the browser blocks the request silently with a network error.

**Fix:** Add `https://api.partnercenter.microsoft.com/*` to `host_permissions` in `manifest.json`:

```json
{
  "manifest_version": 3,
  "name": "Partner Center Tenant Domain",
  "version": "1.0",
  "content_scripts": [
    {
      "matches": ["https://partner.microsoft.com/dashboard/v2/customers/granularadminaccess/list*"],
      "js": ["content.js"],
      "run_at": "document_idle"
    }
  ],
  "host_permissions": [
    "https://partner.microsoft.com/*",
    "https://api.partnercenter.microsoft.com/*"
  ]
}
```

After updating the manifest, go to `chrome://extensions`, find the extension, and click **"Update"** (or disable/re-enable it). The browser will prompt to accept the new host permission.

---

### Bug 2 — Row ID extraction is wrong

**Symptom:** Logs show `Row cell injected: he-row-0 -> —`, `Row cell injected: 1 -> —` — numeric indices instead of tenantId GUIDs.

**Cause:** The code is not correctly reading the `id` attribute from the `<tr>` elements. The actual row IDs in the DOM are `id="row-{tenantId}"` (e.g., `id="row-a1b2c3d4-0001-4abc-8def-111122223333"`). The tenantId must be extracted by stripping the `"row-"` prefix.

**Fix:** Replace whatever row-ID logic is currently used with:

```javascript
// Inside the MutationObserver callback and initial row injection:
function getTenantIdFromRow(row) {
  const id = row.getAttribute('id') || '';
  // id format: "row-{tenantId}" e.g. "row-a1b2c3d4-0001-4abc-8def-111122223333"
  if (id.startsWith('row-')) {
    return id.substring(4); // strip "row-" prefix
  }
  return null;
}

function injectCellIntoRow(row) {
  const tenantId = getTenantIdFromRow(row);
  const domain = tenantId ? (domainMap[tenantId] || '—') : '—';
  // ... create and append the <td> with domain value
  dbg('Row cell injected:', tenantId, '->', domain);
}
```

Only process rows where `getTenantIdFromRow()` returns a non-null value (i.e., rows with the `row-{guid}` id pattern). Skip header rows, loading rows, etc.
